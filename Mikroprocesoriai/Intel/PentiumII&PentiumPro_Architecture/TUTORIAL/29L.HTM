В случае правильного предсказания ветвления буфером адресов ветвления команда ветвления задерживается на один такт, необходимый для предварительной выборки.

Если выполнение команд, находящихся между ветвлениями, может быть завершено за меньшее число тактов, чем необходимо для предварительной выборки следующей команды ветвления, избежать задержки можно одним из следующих способов:

  • Вставка дополнительных команд между командами ветвления, условные переходыпо которым будут выполнены с высокой вероятностью.
  • Раскрутка цикла в случае, когда ветвление обеспечивает повторение итераций.

    Цикл, приведенный под заголовком Код до оптимизации, содержит условный переход назад.

    В Оптимизированном коде цикл раскручен три раза, что снижает число ветвлений и связанных с ними задержек при выполнении.
