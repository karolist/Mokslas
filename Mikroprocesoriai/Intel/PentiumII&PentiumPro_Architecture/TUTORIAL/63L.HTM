Использование синонимов может повысить эффективность кода. Оно также делает его менее понятным для компилятора. Если компилятор не может разрешить неоднозначность в коде с указателями, то им не выполняются следующие действия:
 

  • Присваивание значений во время компиляции.
  • Вставка в код команд считывания из памяти, выполняющихся одновременно с командами записи в память
  • Некоторые другие приемы оптимизации.

    • Для устранения неоднозначностей в коде следует:

  • Использовать локальные переменные вместо указателей.
  • Использовать глобальные переменные там, где нельзя использовать локальные.
  • Размещать операции записи в память далеко от операций чтения.

    • Замечания:

  • Использование переменных вместо указателей делает код более машинно-зависимым и снижает его переносимость.
  • Выигрыш от оптимизации, проведенной компилятором, может не покрывать снижение эффективности кода из-за отказа от использования указателей.

    • Нажмите на кнопку Пояснить, чтобы ознакомиться с комментариями.
