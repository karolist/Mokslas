Применение блочной организации циклов эффективно при выполнении операций над большими матрицами, которые не умещаются в кэш-памяти целиком.

В коде до оптимизации значения M, N и L достаточно велики, чтобы:

  • В конце каждой итерации по j требуемые элементы матрицы C скорее всего вытеснены из кэш-памяти.
  • Только один элемент строки матрицы A может быть использован до помещения в кэш следующей строки этой матрицы.

В оптимизированном коде число итераций для матрицы C уменьшено в четыре раза, и для каждой строки матрицы A последовательно используются четыре ее элемента.

Нажмите на кнопку "Показать", чтобы посмотреть, как работает кэш-память при выполнении каждого из вариантов кода.
